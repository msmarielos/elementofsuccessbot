# Telegram Bot для продажи подписок

Telegram-бот на TypeScript для продажи подписок с интеграцией платежной системы.

## Возможности

- 📋 Просмотр доступных планов подписки
- 💳 Покупка подписки через T‑Bank (интернет‑эквайринг, сценарий WebView / `PaymentURL`)
- ✅ Проверка статуса подписки
- 🔄 Автоматическая активация подписки после оплаты
- 💬 Удобный интерфейс с inline-кнопками

## Требования

- Node.js 18+ 
- npm или yarn
- Telegram Bot Token (получить у [@BotFather](https://t.me/BotFather))

## Установка

1. Клонируйте репозиторий или скачайте файлы проекта

2. Установите зависимости:
```bash
npm install
```

3. Создайте файл `.env` в корне проекта и заполните переменные окружения:
```env
BOT_TOKEN=your_bot_token_here
BOT_USERNAME=your_bot_username

PUBLIC_BASE_URL=https://your-domain.com

TBANK_TERMINAL_KEY=your_terminal_key
TBANK_PASSWORD=your_terminal_password
```

## Настройка

### Получение Bot Token

1. Откройте Telegram и найдите [@BotFather](https://t.me/BotFather)
2. Отправьте команду `/newbot`
3. Следуйте инструкциям для создания бота
4. Скопируйте полученный токен в файл `.env`

### Настройка платежной системы (T‑Bank)

Инструкция по настройке переменных и возвратов: `TBANK_SETUP.md`.

### Настройка планов подписки

Планы подписки настраиваются в файле `src/types/subscription.ts`:

```typescript
export const SUBSCRIPTION_PLANS: SubscriptionPlan[] = [
  {
    id: 'basic',
    name: 'Базовый',
    description: 'Базовый план подписки',
    price: 299,
    currency: 'RUB',
    duration: 30,
    features: ['Доступ к базовым функциям', 'Поддержка 24/7']
  },
  // Добавьте свои планы...
];
```


```bash
npm run build
```

### Запуск в продакшене
```bash
npm start
```

## Использование

После запуска бота пользователи могут:

1. Отправить `/start` - начать работу с ботом
2. Отправить `/plans` - посмотреть доступные планы
3. Отправить `/buy` - купить подписку
4. Отправить `/my_subscription` - проверить статус подписки
5. Отправить `/help` - получить справку

## Проверка оплаты

Бот показывает кнопку **«✅ Проверить оплату»**. Проверка делается через метод T‑Bank **GetState**, как рекомендуется в статье.

## Расширение функционала

### Добавление базы данных

Для хранения подписок в базе данных:

1. Установите ORM (например, TypeORM или Prisma)
2. Создайте модели для подписок
3. Обновите `SubscriptionService` для работы с БД

### Добавление админ-панели

Создайте команды для администраторов:
- Просмотр всех подписок
- Ручная активация подписки
- Статистика продаж
